## Decisions
One of the most important decisions I made during this project was to fully commit to the “Project Archives” concept and treat my portfolio like a library instead of a standard website. I wanted the experience to feel immersive and organized, where each project acts like a book someone can explore. This influenced both the visual design and how I structured the content. Another key decision was improving interactivity, especially by adding the book opening and page flipping effects. In P01, the design looked good but felt too static, so I focused on making it more engaging in P02. I also chose to implement a light and dark mode toggle using CSS variables, which helped maintain the gothic theme while improving usability. Additionally, I made deliberate changes based on peer feedback, such as refining wording, improving titles, and adding a subheading to better explain the site’s purpose. Using GitHub Pages remained an important decision because it allowed me to easily manage updates and track progress between versions.

## What Worked?
What worked best was how the concept translated into a more polished and interactive experience. The library/archive idea became clearer in P02, especially with the improved bookshelf interactions and more refined layout. The addition of animations like book opening and page flipping made the site feel more dynamic and aligned with my original vision. The light and dark mode toggle also worked well, both visually and functionally, and made the site feel more complete. Another success was improving consistency across all the pages such as spacing, typography, and layout feel more intentional compared to P01. I’m also satisfied with how much clearer the content became after revisions. Acting on peer feedback made a noticeable difference in readability and overall professionalism. Finally, continuing to use GitHub and GitHub Pages helped streamline my workflow and made it easy to deploy updates quickly.

## What I would do differently?
If I started over, I would spend more time planning both the structure and interactivity of the site earlier in the process. In P01, I focused mostly on layout, which meant I had to go back later and add more complex interactions like animations. Planning those features earlier would have made the process smoother and the code more organized. I would also prioritize mobile design from the beginning instead of adjusting it later, since issues like excessive scrolling and text size took extra time to fix. Another thing I would change is testing more frequently across different screen sizes and devices to catch inconsistencies sooner. In terms of scope, I would be more realistic about what I could achieve and avoid considering overly complex features that don’t add much to the user experience.

## What I learned?
This project reinforced that building for the web is not just about making something look good. It’s about how clearly it communicates and how it feels to use. Small details like navigation, spacing, and text clarity have a big impact on the overall experience. I also learned that interactivity needs to be planned early, not added later as an afterthought. From a technical perspective, I became more comfortable using HTML, CSS, and JavaScript together, especially with features like CSS variables for theming and JavaScript for interactive elements like menus and animations. I also gained more confidence using GitHub as part of my workflow, particularly for version control and deployment. About myself, I learned that I tend to focus on getting something working first and refining it later, which can be effective but also creates extra work if not balanced with planning. Overall, I learned that iteration is essential! My project improved the most when I took feedback seriously and made intentional revisions rather than settling for the first version.
